    StringBuffer与StringBuilder

    默 2/8/2022 java

    # StringBuffer与StringBuilder

    # 一、引言

    # 由于java中字符串是不可变的常量，存储在方法区的字符串常量池中。如果频繁进行字符串拼接，每一次都会产生新的字符串，造成大量的方法区内存空间浪费

    如果以后需要进行大量的字符串拼接，建议不要用"+"，而是用JDK中自带的：

    java.lang.StringBuffer

    java.lang.StringBuilder

    # 二、常用方法

    # 1、创建一个字符串缓冲区对象（默认初始化容量为16的byte[]数组）

    append()方法在追加字符串时，如果byte数组满了会自动进行扩容（数组拷贝）

    StringBuffer strbuf = new StringBuffer();
    1

    # 2、append() ：拼接字符串

    strbuf.append("a");
strbuf.append(true);
strbuf.append(100L);
    1
    2
    3

    StringBuffer字符串可改变原因：

    • String类底层中的byte[]数组是用final修饰的（注意：在java8及以前是char[]数组）,且数组一旦创建，长度不可改变
    • StringBuffer类底层中的byte[]数组没有被final修饰，所以可以指向新的字符串（拷贝扩容String.arraycopy()），同时在初始化时可以指定数组长度

    # 三、优化StringBuffer性能

    在创建StringBuffer对象时，尽可能预估一下大小，给定一个大一些的初始化容量，以减少扩容的次数

    StringBuffer strbuf = new StringBuffer(100);	//给定初始容量为100
    1

    # 四、StringBuffer与StringBuilder的区别

    StringBuffer中的方法：都有关键字synchronized修饰，表示StringBuffer在多线程环境下运行是安全的

    StringBuilder中的方法：都没有关键字synchronized修饰，表示StringBuilder在多线程环境下运行是不安全的
